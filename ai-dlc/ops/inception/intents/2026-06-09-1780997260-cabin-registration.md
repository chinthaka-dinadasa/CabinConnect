# Intent: Cabin Registration

**Status:** Elaborated
**Date:** 2026-06-09
**Owner:** Chinthaka

---

## What

A Host can register a new cabin on CabinConnect by filling out a form on the frontend with the cabin's name, description, location, base nightly rate, and maximum guest capacity. On submission the cabin is saved via the .NET API, associated with the authenticated Host, and immediately available as a listable unit for future booking workflows.

## Why

Without cabins in the system there is nothing to book. This is the first domain feature that makes CabinConnect functional — the backend scaffold is in place and the first meaningful host-facing capability is to put a cabin on the platform.

## Success Looks Like

- A Host can navigate to a "Register Cabin" page, fill in the required fields, and submit the form
- The cabin is saved to the database and associated with the authenticated Host's user ID
- After successful submission the Host receives a confirmation that the cabin was registered
- The cabin record is retrievable via the API by its ID
- An unauthenticated request to create a cabin is rejected with `401 Unauthorized`
- A Host cannot register a cabin on behalf of another Host

## Assumptions

- All authenticated users are treated as Hosts for this intent — a separate Host role or approval flow is not in scope
- Cabin images are not included — image upload is deferred to a future intent
- A cabin is active/listable immediately on creation — no moderation or approval step
- Base rate is a required decimal per-night price — no minimum or maximum enforced beyond being a valid positive number
- Maximum guest capacity is required with a minimum value of 1
- Location is a free-text string (address/area name) — no geocoding or map integration
- The `cabins` table does not exist yet and will be created as part of this intent (migration included)
- RLS policies will be added to the `cabins` table: Hosts can insert their own cabins; reads are public (needed for browsing in a future intent)

## Open Questions

- None.

## Out of Scope

- Cabin image upload or gallery management
- Editing or deleting a cabin after registration
- Seasonal rates and blackout date management (separate intents)
- Cabin listing and browsing (separate intent)
- Host profile or Host-specific role/permission model
- Multi-currency pricing
- Geocoding or map-based location input
- Cabin availability management
