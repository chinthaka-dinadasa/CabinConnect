## What Didn't Go Well

- **`dotnet ef database update` failed on first attempt** — the fail-fast `Program.cs` pattern (`?? throw`) threw `CORS_ALLOWED_ORIGINS not set` before EF Core could resolve the DbContext. Required a follow-up `IDesignTimeDbContextFactory` to fix. This pattern will recur on every future migration command if not standardised.

- **`.env` connection string truncated silently** — `DATABASE_URL=Host=...;SSL Mode=Require` without double-quotes caused bash to treat `;` as a command separator. `echo $DATABASE_URL` only returned `Host=db.xxx.supabase.co`. Took two diagnostic steps to identify (first suggested `xargs`, then `source`, then found the quoting issue). Silent failures like this are expensive to debug.

- **`IDesignTimeDbContextFactory` not part of the backend scaffold unit** — it was generated reactively after `dotnet ef` failed rather than proactively during the scaffold bolt. The scaffold bolt's purpose was to establish all foundational patterns, and design-time tooling support is foundational.

## Post-Retro Improvement Workflow

### Step 1 — Improvement proposals

Two findings from this retro point to fixable rules/guidelines:

**Proposal 1 — IDesignTimeDbContextFactory standard**
- **File:** `ai-dlc/rules/code-standards.md` (or a new .NET-specific checklist entry)
- **Finding:** When `Program.cs` uses fail-fast env var checks, `dotnet ef` design-time commands fail unless `IDesignTimeDbContextFactory<TDbContext>` is present in `Data/`
- **Proposed addition:** Add to the .NET "Always do these" section: *"When `Program.cs` uses fail-fast env var validation, include `IDesignTimeDbContextFactory<TDbContext>` in `Data/` so EF Core design-time commands bypass application startup."*
- **Why it prevents recurrence:** Future scaffold bolts will include the factory from day one instead of adding it reactively after a failed migration command

**Proposal 2 — .env quoting standard**
- **File:** `backend/.env.example`
- **Finding:** Connection strings containing `;` or spaces silently truncate when sourced without double-quotes, producing hard-to-diagnose partial values
- **Proposed change:** Update the `DATABASE_URL` example line to use double-quotes: `DATABASE_URL="Host=...;Port=5432;Database=postgres;Username=postgres;Password=...;SSL Mode=Require"` and add a comment: `# Values containing ; or spaces must be double-quoted`
- **Why it prevents recurrence:** Engineers see the correct format from the first time they copy the example

Awaiting your approval, rejection, or revision of each proposal before creating the improvement files and applying changes.
